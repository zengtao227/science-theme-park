# Proposal: Math Visualization Improvements
**Status**: DRAFT  
**Date**: 2026-02-09  
**Author**: Trae (Acting Director)  

## 1. Executive Summary
响应关于数学模块（二项式、勾股定理、直线函数）缺乏直观性、场景应用牵强和交互枯燥的反馈。本提案利用 NVIDIA AI 模型生成的创意，提出了三个全新的游戏化改进方案，旨在通过物理直觉和真实场景来教授抽象数学概念。

---

## 2. Module A: The Binomial Factory (`sm2-01`)
**当前问题**: "Architect/Scrapper" 场景与公式 `(a+b)²` 的联系过于牵强，学生无法理解为何计算面积。
**改进目标**: 引入真实且自然的 `(a+b)²` 应用场景。

### 推荐方案: 孟德尔的遗传花园 (Mendel's Garden)
**核心概念**: 将 `(a+b)² = a² + 2ab + b²` 映射到生物遗传学中的基因组合 `(A+a)² = AA + 2Aa + aa`。
*   **场景**: 玩家管理一个未来植物实验室。
*   **任务**: 培育具有特定性状的"量子豌豆"。你需要混合显性基因 (A) 和隐性基因 (a)。
*   **交互**:
    *   输入基因池比例（例如 A=3, a=2）。
    *   可视化：展示杂交后的子代分布。
        *   **AA (a²)**: 纯显性（如发红光）。
        *   **2Aa (2ab)**: 杂合子（如发紫光）。
        *   **aa (b²)**: 纯隐性（如发蓝光）。
*   **数学联系**: 直观看到 `2ab` 项为何系数是 2（因为 Aa 和 aA 是同一种表现型但来源不同）。

### 备选方案: 精密工程误差分析
*   **场景**: 制造高精度正方形芯片。
*   **任务**: 目标边长 `a`，但存在加工误差 `b`。
*   **交互**: 调整误差 `b`，观察总面积 `(a+b)²` 中，误差项 `2ab + b²` 如何导致废品率激增。直观理解为何微小线性误差会带来"平方级"的面积代价。

## 3. Module B: The Pythagorean Studio (`sm2-02`)
**当前问题**: 可视化被截断，静态方块网格枯燥，“数格子”式学习效率低。
**改进目标**: 实现动态、即时反馈的面积守恒演示。

### 推荐方案: 动态流体室 (The Fluid Chamber)
**核心概念**: 利用物理引擎模拟水流，直观展示体积/面积守恒。
*   **技术**: 使用 **Matter.js** (2D物理引擎) 或自定义粒子系统。
*   **交互**:
    *   屏幕上有三个旋转连接的正方形容器（a, b, c）。
    *   初始状态：a 和 b 容器装满“能量液”。
    *   **操作**: 玩家旋转整个装置。
    *   **反馈**: 液体受重力作用流向 c 容器。
    *   **结果**: 刚好填满 c 容器，直观证明 `a² + b² = c²`，无需任何数字计算。
    *   **解决痛点**: 彻底摆脱“数格子”，用物理直觉替代。

### 备选方案: Perigal 的动态剪切
*   **交互**: 自动/手动演示将 b² 剪切成4块，与 a² 一起完美拼入 c² 的动画。使用 SVG 路径变形动画 (Framer Motion)。

## 4. Module C: Linear Navigator (`sm2-03`)
**当前问题**: “激光反射”玩法让学生觉得在凑数，无法理解 m (斜率) 和 c (截距) 的几何意义。
**改进目标**: 赋予 m 和 c 明确的物理/几何操作反馈。

### 推荐方案: 极速滑雪/飞行模拟 (Slope & Altitude)
**核心概念**: 将 m 映射为**坡度/爬升率**，将 c 映射为**起跳高度/初始海拔**。
*   **场景**: 设计滑雪跳台或飞机起飞路径。
*   **任务**: 让滑雪者/飞机安全穿过一系列空中的圆环（坐标点）。
*   **交互**:
    *   **调整 m (Slope)**: 改变滑雪板/飞机的**角度**。直观感受 m > 0 (上坡/起飞)、m < 0 (下坡/俯冲)、|m| 越大越陡。
    *   **调整 c (Intercept)**: 改变起跳台的**高度**。
*   **视觉反馈**:
    *   调整参数时，即时预览轨迹线。
    *   播放时，角色沿线运动，包含物理反馈（速度随斜率变化）。
*   **解决痛点**: 学生能直接感受到“m 控制方向/陡峭度”，“c 控制位置高低”。
