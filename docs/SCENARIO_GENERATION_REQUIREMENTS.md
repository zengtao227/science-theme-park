# 场景生成要求（交接版）

项目路径说明：以下路径均指向 projects 中的“瑞士中学数理化生物练习项目”。

## 1) 适用范围
- 适用于 Math/Physics/Chemistry/Biology 模块中的题干（prompt/mission/scenario）。
- 适用于新题生成与旧题重写（尤其是 SM 系列场景化升级）。

## 2) 每道题的强制结构
每道题必须同时具备以下四项：
1. 背景：具体人物与情境（生活/校园/工作/实验室/工程）。
2. 任务：明确“要算什么”，并说明“为什么要算”。
3. 已知条件：给定量必须带单位/来源，不允许裸参数（如仅 a=35,b=120）。
4. 结果用途：计算结果将用于何种决策或行动。

## 3) 写作规范
- 优先使用生活化与职业化场景（消防、物流、医疗、建筑、交通、实验室等）。
- 初中生可理解：避免只给公式指令；先讲问题，再引出计算。
- 数值与单位完整一致（m, m^2, m^3, s, kg, CHF 等）。
- 同一题干中，场景目标与最终求解量必须一致。

## 4) 反例与正例
- 反例（不合格）：
  - “消防救援：a=35,b=120 求C”
- 正例（合格）：
  - “消防员接警后，需把梯子从消防车架到四楼窗台。窗台距地 9m，消防车停在离楼 8m 的安全区。请计算梯子至少长度，以判断现有 12m 梯是否可用。”

## 5) 技术落地约束（与渲染规则保持一致）
- prompt 文本走 i18n，不回退硬编码英文。
- LaTeX 字段保持双反斜杠规范（避免 `60imes360imes3` 类转义错误）。
- 提交前必须通过：
  - `npm run lint`
  - `npm run build`
  - `npm run validate:translations`（对应学科）
  - `bash scripts/audit-rendering.sh`
