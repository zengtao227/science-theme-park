# GM3.01 难度重新设计 - 概念深度递进

## 问题分析

当前设计的问题：
1. ❌ 只是数量增加（6→52→100→500），没有概念深度
2. ❌ ELITE级别500个方块显示不下，数字溢出
3. ❌ 所有难度都是同一个公式 P(E) = favorable/total，没有思维挑战

## 新设计原则

**难度递进应该体现：**
1. 概念理解的深度
2. 问题复杂度的增加
3. 需要的数学技巧
4. 思维方式的转变

---

## BASIC_PROB 阶段重新设计

### BASIC 难度：直接观察样本空间
**概念**：样本空间直接可见，一一对应
**例子**：
- 掷一个骰子，6个面直接看到
- 抛一个硬币，2个面直接看到
- 从6张牌中抽1张特定的牌

**可视化**：6-10个方块，直接显示

### CORE 难度：理解"等可能性"和"组合"
**概念**：需要理解不同排列是不同结果
**例子**：
- 掷两个骰子，(1,2)和(2,1)是不同的 → 36种可能
- 从一副牌中抽牌，需要理解花色和点数的组合
- 理解"至少一个"需要用补集思维

**可视化**：显示组合的概念，如两个骰子的36种组合

### ADVANCED 难度：条件概率的隐含应用
**概念**：有条件限制的概率，需要重新定义样本空间
**例子**：
- "已知第一次抽到红球，第二次抽到红球的概率"（不放回）
- "在偶数点出现的条件下，点数大于4的概率"
- 需要理解样本空间会因条件而改变

**可视化**：显示条件如何缩小样本空间

### ELITE 难度：复合事件和多步推理
**概念**：需要分解成多个事件，使用加法/乘法原理
**例子**：
- "至少有一个6出现"的概率（需要用1减去"都不是6"）
- "恰好有2个红球"的概率（需要考虑不同的抽取顺序）
- 需要多步推理和事件分解

**可视化**：显示事件树或分解步骤

---

## BINOMIAL 阶段重新设计

### BASIC 难度：理解二项分布的基本概念
**概念**：固定次数，固定概率，计算恰好k次
**例子**：
- n=3, k=2, p=0.5（抛3次硬币，恰好2次正面）
- 直接套用公式

**可视化**：显示3次试验的8种可能结果

### CORE 难度：理解二项系数的意义
**概念**：理解C(n,k)代表"选择的方式数"
**例子**：
- n=5, k=3, p=0.5
- 需要理解为什么是C(5,3)=10种方式

**可视化**：显示10种不同的"3次成功"的排列

### ADVANCED 难度：非对称概率 + 期望值
**概念**：p≠0.5时的计算，理解偏态分布
**例子**：
- n=8, k=5, p=0.6（不对称）
- 或者计算期望值 E(X) = np

**可视化**：显示偏态的分布图

### ELITE 难度：累积概率和逆向问题
**概念**：计算"至少k次"或"至多k次"，或反推n
**例子**：
- "至少成功6次的概率" = P(X≥6) = 1 - P(X≤5)
- "要使成功概率>0.9，至少需要多少次试验？"

**可视化**：显示累积分布

## CONDITIONAL 阶段重新设计

### BASIC 难度：理解条件概率的定义
**概念**：P(A|B) = P(A∩B) / P(B)
**例子**：
- 直接给出P(A), P(B), P(A∩B)，套用公式
- 数值简单，如0.5, 0.6, 0.3

**可视化**：文氏图，清晰显示交集

### CORE 难度：从问题中提取条件
**概念**：需要从文字描述中识别条件和事件
**例子**：
- "一副牌中，已知抽到红色牌，求是红心的概率"
- 需要自己识别：B="红色"，A="红心"，A∩B="红心"

**可视化**：显示如何从问题提取信息

### ADVANCED 难度：贝叶斯思维
**概念**：P(A|B)和P(B|A)的关系，贝叶斯公式
**例子**：
- "已知测试阳性，真正患病的概率"（需要考虑假阳性率）
- 需要理解P(病|阳性) ≠ P(阳性|病)

**可视化**：显示贝叶斯翻转的概念

### ELITE 难度：多重条件和独立性
**概念**：P(A|B∩C)，或判断事件独立性
**例子**：
- "已知两个条件同时满足，求第三个事件的概率"
- 或"判断A和B是否独立"（P(A|B) = P(A)?）

**可视化**：显示多重条件的关系

## 具体实现建议

### 1. BASIC_PROB 新数据结构

```typescript
// BASIC: 直接样本空间
{ id: "BP_B1", favorable: 1, total: 6, type: "direct", context: "single_die_one" }

// CORE: 组合样本空间
{ id: "BP_C1", favorable: 6, total: 36, type: "combination", context: "two_dice_sum_7", 
  explanation: "需要理解(1,6),(2,5),(3,4),(4,3),(5,2),(6,1)是6种不同的结果" }

// ADVANCED: 条件概率（隐含）
{ id: "BP_A1", favorable: 3, total: 6, type: "conditional", context: "die_even_and_gt4",
  explanation: "在偶数(2,4,6)的条件下，大于4的只有6，所以是1/3而不是1/6" }

// ELITE: 复合事件
{ id: "BP_E1", type: "compound", context: "at_least_one_six",
  explanation: "至少一个6 = 1 - P(都不是6) = 1 - (5/6)^2" }
```

### 2. 可视化调整

**BASIC**: 显示6-10个大方块
**CORE**: 显示组合矩阵（如6×6的骰子组合）
**ADVANCED**: 显示条件如何筛选样本空间
**ELITE**: 显示事件树或补集思维

### 3. 问题文本重写

每个问题需要：
1. 清晰的情境
2. 明确的数据
3. 引导思考的提示（ADVANCED和ELITE）
4. 为什么这个难度需要这种思维方式
