# Claude 全局工作流程规范 (Claude Global Workflow Guidelines)

本文件定义 Kiro 以及本项目中所有 AI 必须遵循的强制工作流程。

## 🚨 强制第一性原理分析框架 (Mandatory First-Principles Analysis)

**在提出任何修改方案之前，必须显式输出以下分析（不可跳过）：**

### 📋 必须输出的分析框架

每次提出方案前，必须在回复中包含以下结构化分析：

```
## 第一性原理分析

### 1. 问题本质
- 这个问题的根本原因是什么？（不是表面症状）
- 为什么会出现这个问题？

### 2. 现有机制审查
- 系统中是否已有相关机制可以复用？
- 如果有，为什么不能直接使用？

### 3. 约束条件
- 硬约束：[不可违反的规则]
- 软约束：[可调整的参数]

### 4. 方案评估
- 方案A: [描述] - 优点/缺点
- 方案B: [描述] - 优点/缺点（如果有）
- 推荐方案及理由

### 5. 最小化检查
- 这个方案是否是最简单的？
- 能否用更少的代码实现？
- 是否在添加代码？如果是，有没有办法通过删除或修改现有代码来解决？

### 6. 副作用检查
- 这个修改会影响哪些其他模块？
- 是否会引入新的问题？
```

**Remember**: 最好的代码是没有代码，次好的是更少的代码

## 🔍 修复后全面检查规范 (Comprehensive Post-Fix Inspection)

**修复任何问题后，必须对整个代码库进行相似问题的全面检查！**

### 检查流程

1. 修复一个具体问题
2. 分析问题的本质模式
3. 在整个代码库中搜索相似模式
4. 修复所有发现的相似问题
5. 运行代码检查验证

### 输出格式

修复后必须在回复中包含：

```
## 相似问题检查

### 检查范围
- [检查了哪些文件/模块]

### 检查结果
- ✅ 未发现其他相似问题
- 或
- ⚠️ 发现 N 个相似问题，已修复
```

## 🔧 代码质量检查 (QA)

**修改代码后，必须运行代码检查验证！**

### 检查内容
- ❌ 语法错误
- ❌ 未定义变量
- ❌ 类型错误

## 📝 Git 提交规范

- `refactor:` 重构代码
- `perf:` 性能优化
